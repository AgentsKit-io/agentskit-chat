**One agent experience. Every interface.**

In the age of AI, chat is not a single React widget. It is the product surface your users meet on
the web, in a desktop shell, on mobile, and in the terminal. Rebuilding that experience per framework
is how teams ship drift: different tools, different policy, different bugs.

AgentsKit Chat is the **application framework** for that problem. You define routes, actions,
components, sessions, and backend seams once. Then each AgentsKit UI binding renders the same
signature natively — React, Vue, Svelte, Solid, Angular, React Native, and Ink.

## Start in sixty seconds

Pick a renderer. The CLI scaffolds a shared definition, Web-standard handler, native shell, and test
without copying an AgentsKit runtime.

<InstallCommand />

```bash
cd my-chat
pnpm install
pnpm test
```

## Why this is the go-to chat framework

| You need | What Chat gives you |
| --- | --- |
| One product brain | A framework-neutral `defineChat` definition shared by every shell |
| Platform-native UX | Renderers over real AgentsKit bindings — not a fake “universal” DOM |
| Safe agent UI | Closed component registry + action policy outside the model |
| Deterministic workflows | Routes and conversation state machines when you refuse to gamble on the LLM |
| Growth path | Sessions, theming, server handlers, Ask backends, traces — without rewriting day one |

Missing framework-neutral primitives are fixed in [AgentsKit](https://www.agentskit.io/docs) first.
Chat owns **composition and versioned application boundaries**, not a second controller.

## The shape of a Chat app

```ts
import { defineChat } from '@agentskit/chat'
import type { AdapterFactory } from '@agentskit/core'

export const createSupportChat = (adapter: AdapterFactory) =>
  defineChat({
    id: 'support',
    chat: {
      adapter,
      systemPrompt: 'Help users understand the product.',
    },
  })
```

Render the same definition on any supported shell:

<FrameworkTabs defaultValue="react">
  <Framework name="react">

```tsx
import { AgentChat } from '@agentskit/chat/react'
import { createSupportChat } from './support-chat'
import { adapter } from './adapter'

export const Support = () => (
  <AgentChat
    definition={createSupportChat(adapter)}
    placeholder="Ask about the product"
  />
)
```

  </Framework>
  <Framework name="vue">

```vue
<script setup lang="ts">
import { AgentChat } from '@agentskit/chat/vue'
import { createSupportChat } from './support-chat'
import { adapter } from './adapter'

const definition = createSupportChat(adapter)
</script>

<template>
  <AgentChat :definition="definition" placeholder="Ask about the product" />
</template>
```

  </Framework>
  <Framework name="ink">

```tsx
import { render } from 'ink'
import { AgentChat } from '@agentskit/chat/ink'
import { createSupportChat } from './support-chat'
import { adapter } from './adapter'

render(
  <AgentChat
    definition={createSupportChat(adapter)}
    placeholder="Ask about the product"
  />,
)
```

  </Framework>
</FrameworkTabs>

Streaming, tools, memory, retry, editing, regeneration, confirmation, and cancellation remain
AgentsKit behavior. Chat keeps those semantics stable across shells.
