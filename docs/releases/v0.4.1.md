# AgentsKit Chat 0.4.1

`0.4.1` is a metadata-only patch that republishes both fixed-group packages with
the canonical documentation homepage. Runtime APIs, protocol meaning, component
contracts, and renderer behavior are unchanged from `0.4.0`.

## What changed

- Both `@agentskit/chat` and `@agentskit/chat-cli` publish
  `homepage: https://chat.agentskit.io/docs`.
- Packaged release artifacts fail closed unless that homepage is present.
- Repository, bugs, license, provenance, and export maps remain the `0.4.0`
  contracts.

## Why this patch exists

`0.4.0` was published before the docs-portal homepage landed in package
manifests. Registry metadata for exact `0.4.0` still points at the GitHub
readme URL. npm does not allow rewriting an already-published version, so this
patch is the only way to align public package pages with the production docs
portal.

## Upgrade

Update both fixed-group packages together:

```bash
npm install @agentskit/chat@0.4.1
npm install --save-dev @agentskit/chat-cli@0.4.1
```

No application migration is required. Hosts pinned to `^0.4.0` receive this
patch through ordinary semver ranges after publish.

## Evidence

Clean external consumption of exact `0.4.0` was verified before preparing this
patch: both packages install from npm, root ESM and CommonJS entrypoints load,
all declared export targets exist, and CLI scaffolding produces a React project
that typechecks, tests, and builds. See
[clean-install-0.4.0 evidence](./clean-install-0.4.0.md). Post-publication
verification of the exact patch is recorded in
[clean-install-0.4.1 evidence](./clean-install-0.4.1.md).
